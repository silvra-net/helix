The post-quantum blockchain.

Helix (HLX) is a Layer-1 blockchain written in Rust. Every signature uses NIST FIPS 204 (ML-DSA-65), so the ledger stays secure even against a quantum computer. Proof-of-stake, BFT consensus, two-second blocks. Made in Germany 🇩🇪.

Get started

Send your first HLX in three steps

Everything runs from one binary, helix. Download it for your OS from the latest release (linux-x86_64, macos-arm64, windows-x86_64). Out of the box it talks to this network — no configuration needed.

1Make a wallet

Creates a key and prints a 24-word recovery phrase. Write the phrase down once — it is the only backup.

helix wallet new

2Check your balance

Look up any address — yours or someone else's — for balance, stake and history.

helix account <your-address>

3Send some HLX

Pay a raw address or a name.hlx. The fee is priced from the chain automatically.

helix tx send <to> <amount>
helix tx status <hash>

Lost a wallet? helix wallet restore rebuilds it from the 24 words. Point any command at a different node with --node <url> — the default is https://helix.silvra.net.

Run a node

Join the network

A node with no configuration joins this chain automatically: it fetches genesis from a seed peer, recomputes the genesis state hash itself before trusting anything, syncs the history, and then follows live. 

# download the binary, then just:
helix start

Running from behind a home connection or Cloudflare? Helix speaks libp2p over WebSocket, so a node can gossip through an HTTPS tunnel on port 443 — no open inbound port required. See the README for the reverse-proxy setup.

Want your own private chain instead of joining this one? Start with HELIX_NEW_CHAIN=1 for a standalone genesis.

Becoming a validator

  • Stake at least 100,000 HLX to enter the validator set.
  • Don't have a machine to run? Delegate to a validator and earn a share of the rewards, auto-compounded.
  • Unstaking has a 7-day unbonding period; double-signing is slashed 5% and jailed immediately.
helix tx stake <amount>
helix tx delegate <validator> <amount>
Tokenomics

How HLX is issued

Maximum supply33,000,000 HLX (a hard cap)
Block reward1 HLX per block, halving every 15,768,000 blocks (~1 year)
Block time~2 seconds
Transaction feeEIP-1559 base fee (per byte, burned) + tip to the validator
Min. validator stake100,000 HLX
Unbonding period7 days
Slashing5% + immediate jail for double-signing

There was no pre-mine dumped to a founder wallet. The supply enters circulation gradually through block rewards, Bitcoin-style — a halving schedule under a fixed 33M cap.

Fees follow Ethereum's model: a per-byte base fee is burned (removing HLX from supply), and anything above it tips the validator. Higher demand burns more, so the money supply is disinflationary by design.

Staking, delegation and governance are three separate rights: delegators earn yield but governance votes stay with self-stakers. Validator commission is capped at 50%.

Technology

What's under the hood

Post-quantum crypto

Signatures are ML-DSA-65 (NIST FIPS 204); SLH-DSA (FIPS 205) is available as a hash-based alternate. Hashing is BLAKE3. No home-rolled crypto — only vetted crates.

BFT consensus

Proof-of-stake with a Tendermint-style BFT engine: 2/3+1 quorum per round, cross-round vote locking for safety, and deterministic slashing evidence on-chain.

Smart contracts

Deterministic WASM (floats disabled by construction), fuel-metered, with host imports for storage and value transfer — no cross-contract calls, so reentrancy can't exist.

Human-readable names

Register a name.hlx and receive to it instead of a raw address. Resolvable across the network and from the wallet.

Accounts that recover

Social recovery via guardians, proof-of-personhood for sybil resistance, and on-chain governance over protocol parameters.

Honest by default

Every transaction carries a verifiable receipt; the node exposes a deterministic state hash so any two nodes can prove they computed the same ledger.
